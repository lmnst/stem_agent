# pybugs benchmark

40 small Python single-function bug-repair tasks. Each task is one folder
containing:

- `solution.py`: a module with a buggy function.
- `test_solution.py`: pytest cases that pass on the correct
  implementation and fail on the buggy one.

A task is **solved** when every `test_*` callable returns without
raising, run on a temp copy of the task folder.

## Splits

- `train/`: 12 tasks. Used by domain analysis (uniform-priority probe)
  to estimate primitive priors and the recommended budget.
- `dev/`: 8 tasks. Used by generational evolution to score candidate
  blueprints.
- `test/`: 12 tasks. Held out; never opened during analysis or
  evolution. The headline pass-rate numbers come from this split.
- `challenge/`: 8 tasks. Bugs whose repair is not a single-site
  application of any primitive in the bank. Each task represents a
  real bug class. Both the stem and the deployed evolved blueprint
  fail every challenge task; the split exists to bound where the
  primitive bank breaks down. Reported separately from the in-bank
  result, with actual attempts and effective-budget attempts shown
  side by side.

The original 32-task split (train/dev/test) keeps the eight in-bank
families roughly proportionate, with one deliberate exception: the
equality family appears only in the test split, to test whether
evolution overfits to the training prior.

## In-bank bug families (train + dev + test)

- comparator-strict (`<` vs `<=`, `>` vs `>=`)
- comparator-flip (`<` vs `>`, `<=` vs `>=`)
- equality (`==` vs `!=`)
- integer-constant +/-1
- arithmetic-op swap (`+` `-` `*` `/` `//`)
- boolean-op swap (`and` vs `or`)
- `True` vs `False`
- swapped first-two positional call args

## Out-of-bank bug families (challenge)

Each task is named for a real bug class; the bug class is the spec
written before any code. The repair requires more than a single
single-site AST mutation, so the primitive bank cannot reach it.

- `ch_001`: wrong identifier in a sum (`a + a + c` should be
  `a + b + c`).
- `ch_002`: constant scaled by the wrong factor (`* 100` should be
  `* 5`); the delta is larger than what `shift_const_pm1` can reach.
- `ch_003`: multi-edit arithmetic and constant sign (`a + b - 1`
  should be `a - b + 1`).
- `ch_004`: missing branch (no `if b == 0` guard before division).
- `ch_005`: off-by-one upper bound on a `range`-driven loop sum
  (`range(1, n)` should be `range(1, n + 1)`).
- `ch_006`: wrong dictionary key (reads `d['celsius']` when it
  should read `d['fahrenheit']`); a string-literal repair the
  primitive bank does not perform.
- `ch_007`: wrong normalization (divides by `len(xs)` instead of
  `sum(xs)`); a function-name repair the primitive bank does not
  perform.
- `ch_008`: multi-site arithmetic restructuring
  (`price - tax_rate * 1` should be `price * (1 + tax_rate)`).

Each challenge task has at least four variants generated by the
primitive bank, so the failure mode is a real attempt-and-fail, not
a no-variant short-circuit. None of the variants passes the task's
tests; the test suites use multiple data points so a coincidental
single-site fix would be caught on at least one case.
